Jcurl

A lightweight, cross-platform HTTP API client — CLI + JavaFX + Swing

JDK 17+ SpringBoot 3.2 JavaFX 21 Swing (Win7+)

Overview

Jcurl is a desktop HTTP API client inspired by Postman, with a curl-compatible CLI mode. It features a modular architecture with three independently buildable modules: a core backend with CLI, a modern JavaFX GUI, and a legacy Swing GUI for Windows 7 compatibility.

Installation

Build from Source

git clone https://github.com/weacsoft/Jcurl.git
cd Jcurl
mvn clean package -DskipTests

CLI Mode (curl-compatible)

The CLI mode supports 30+ curl options, making it a drop-in replacement for most curl use cases.

Basic Usage

# Simple GET
java -jar jcurl-core.jar https://httpbin.org/get

# GET with response headers (-i)
java -jar jcurl-core.jar -i https://httpbin.org/get

# POST with JSON body
java -jar jcurl-core.jar -X POST \
  -d '{"name":"test","value":123}' \
  -H "Content-Type: application/json" \
  https://httpbin.org/post

Authentication

# Basic auth
java -jar jcurl-core.jar -u user:password https://api.example.com/data

# Bearer token
java -jar jcurl-core.jar --bearer "eyJhbGciOi..." https://api.example.com/secure

File Upload

# Multipart form upload
java -jar jcurl-core.jar -F "file=@photo.jpg" -F "name=hello" https://httpbin.org/post

# PUT upload
java -jar jcurl-core.jar -T largefile.zip https://api.example.com/upload

Output Control

# Save to file (-o) + custom format (-w)
java -jar jcurl-core.jar -s -o response.json \
  -w "HTTP %{http_code} | %{time_total}s | %{size_download} bytes\n" \
  https://httpbin.org/get

# Silent mode (-s) + include headers (-i)
java -jar jcurl-core.jar -si https://httpbin.org/get

JavaFX GUI Guide

Launching

Key Features

Syntax Highlighting: JSON, XML, and HTML responses are automatically formatted and color-coded using RichTextFX.

Variable Auto-completion: Type {{ in the URL field or body editor to get a popup with available environment variables and built-in functions.

Cookie Management: Cookies are automatically isolated per collection. Set-Cookie headers are parsed and stored automatically.

Auto Headers: 6 default headers are auto-merged. Each header has a checkbox to individually enable/disable it.

Drag & Drop: Reorder requests and folders in the collection tree by dragging.

Plugin Manager: Install, uninstall, and reload plugins from the UI. Plugin operations run in background threads.

Plugin Development

Jcurl supports 4 extension points. Create a .java file implementing any of them:

1. RequestInterceptor

Modify requests before they are sent. Add headers, log requests, transform bodies.

@Override
public RequestConfig beforeRequest(RequestConfig config, PluginContext ctx) {
    config.getHeaders().add(new Header("X-Request-Id", UUID.randomUUID().toString()));
    ctx.log("info", "Sending to: " + config.getUrl());
    return config;
}

2. ResponseInterceptor

Process responses after they are received. Extract tokens, log metrics, modify headers.

@Override
public ResponseData afterResponse(ResponseData response, RequestConfig config, PluginContext ctx) {
    ctx.log("info", "Got: " + response.getStatusCode() + " in " + response.getTiming().getTotalMs() + "ms");
    return response;
}

3. VariableFunctionExtension

Register custom {{$function}} variables for use in URLs and bodies.

@Override
public List getFunctionNames(PluginContext ctx) {
    return List.of("md5", "base64");
}

@Override
public String executeFunction(String name, String args, PluginContext ctx) {
    if ("md5".equals(name)) return md5(args);
    if ("base64".equals(name)) return Base64.getEncoder().encodeToString(args.getBytes());
    return null;
}

4. MetricsCollectorExtension

Collect custom metrics during performance testing.

@Override
public Map collectMetrics(RequestConfig config, ResponseData response, PluginContext ctx) {
    Map m = new HashMap<>();
    m.put("response_kb", response.getSize() / 1024.0);
    m.put("is_success", response.getStatusCode() < 400 ? 1.0 : 0.0);
    return m;
}
Installation: Copy your .java file to ~/.api-client/plugins/ or use the Plugin Manager dialog. Plugins are compiled at runtime — no pre-compilation needed.

项目简介

Jcurl 是一个受 Postman 启发的桌面 HTTP API 客户端，同时提供 curl 兼容的命令行模式。采用模块化架构，包含三个可独立编译的模块：核心后端（含 CLI）、JavaFX 图形界面、以及兼容 Windows 7 的 Swing 图形界面。

插件开发

Jcurl 支持 4 大扩展点。创建一个 .java 文件实现任意扩展点即可：

1. 请求拦截器 (RequestInterceptor)

在请求发送前修改。可添加请求头、记录日志、转换请求体。

@Override
public RequestConfig beforeRequest(RequestConfig config, PluginContext ctx) {
    config.getHeaders().add(new Header("X-Request-Id", UUID.randomUUID().toString()));
    ctx.log("info", "发送到: " + config.getUrl());
    return config;
}

2. 响应拦截器 (ResponseInterceptor)

在响应接收后处理。可提取 token、记录指标、修改响应头。

@Override
public ResponseData afterResponse(ResponseData response, RequestConfig config, PluginContext ctx) {
    ctx.log("info", "收到: " + response.getStatusCode() + " 耗时 " + response.getTiming().getTotalMs() + "ms");
    return response;
}

3. 变量函数扩展 (VariableFunctionExtension)

注册自定义 {{$function}} 变量，可在 URL 和 body 中使用。

@Override
public List getFunctionNames(PluginContext ctx) {
    return List.of("md5", "base64");
}

@Override
public String executeFunction(String name, String args, PluginContext ctx) {
    if ("md5".equals(name)) return md5(args);
    if ("base64".equals(name)) return Base64.getEncoder().encodeToString(args.getBytes());
    return null;
}

4. 指标采集扩展 (MetricsCollectorExtension)

在性能测试中采集自定义指标。

@Override
public Map collectMetrics(RequestConfig config, ResponseData response, PluginContext ctx) {
    Map m = new HashMap<>();
    m.put("response_kb", response.getSize() / 1024.0);
    m.put("is_success", response.getStatusCode() < 400 ? 1.0 : 0.0);
    return m;
}
安装方式：将 .java 文件复制到 ~/.api-client/plugins/ 目录，或通过插件管理对话框安装。插件在运行时编译，无需预编译。
