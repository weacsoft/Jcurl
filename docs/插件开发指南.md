# JPostman 插件开发指南

JPostman 使用 Java SPI (ServiceLoader) 机制加载插件。每个 JAR 插件使用独立的 URLClassLoader 实现类隔离,支持运行时热加载/卸载。

## 插件接口

所有插件必须实现 `com.jpostman.plugin.JPostmanPlugin` 接口:

```java
public interface JPostmanPlugin {
    void onLoad();           // 插件加载时调用, 用于初始化
    void onUnload();         // 插件卸载时调用, 用于资源释放
    String getName();        // 返回插件名称
    String getVersion();     // 返回插件版本
    String getDescription(); // 返回插件描述
}
```

此外有 3 个可选扩展点接口,插件按需实现:

| 扩展点 | 接口 | 方法 | 触发时机 |
|--------|------|------|----------|
| 请求拦截器 | `RequestInterceptor` | `RequestConfig intercept(RequestConfig config)` | 请求发送前 |
| 响应处理器 | `ResponseProcessor` | `ResponseData process(ResponseData response)` | 收到响应后 |
| 变量函数 | `VariableFunction` | `String getFunctionName()` + `String execute()` | 变量解析时 ({{$funcName}}) |

## 快速开始

### 1. 创建 Maven 项目

```xml
<project>
    <groupId>com.example</groupId>
    <artifactId>my-jpostman-plugin</artifactId>
    <version>1.0.0</version>
    <packaging>jar</packaging>

    <dependencies>
        <!-- JPostman 插件接口 ( provided scope, 运行时由宿主提供 ) -->
        <dependency>
            <groupId>com.jpostman</groupId>
            <artifactId>jpostman</artifactId>
            <version>0.1.0-SNAPSHOT</version>
            <scope>provided</scope>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <configuration>
                    <source>8</source>
                    <target>8</target>
                </configuration>
            </plugin>
        </plugins>
    </build>
</project>
```

### 2. 编写插件类

```java
package com.example;

import com.jpostman.plugin.JPostmanPlugin;
import com.jpostman.plugin.RequestInterceptor;
import com.jpostman.model.dto.RequestConfig;

public class TimestampPlugin implements JPostmanPlugin, RequestInterceptor {

    @Override
    public void onLoad() {
        System.out.println("时间戳插件已加载");
    }

    @Override
    public void onUnload() {
        System.out.println("时间戳插件已卸载");
    }

    @Override
    public String getName() {
        return "Timestamp Plugin";
    }

    @Override
    public String getVersion() {
        return "1.0.0";
    }

    @Override
    public String getDescription() {
        return "自动为请求添加时间戳头 X-Timestamp";
    }

    @Override
    public RequestConfig intercept(RequestConfig config) {
        // 在请求发送前添加时间戳头
        String timestamp = String.valueOf(System.currentTimeMillis());
        config.getHeaders().add(new com.jpostman.model.KeyValue(
            "X-Timestamp", timestamp, "自动时间戳", true
        ));
        return config;
    }
}
```

### 3. 注册 SPI 服务

在 `src/main/resources/META-INF/services/` 下创建文件 `com.jpostman.plugin.JPostmanPlugin`,内容为插件实现类的全限定名:

```
com.example.TimestampPlugin
```

一个 JAR 中可以包含多个插件类,每行一个:

```
com.example.TimestampPlugin
com.example.SignPlugin
com.example.LogPlugin
```

### 4. 打包并安装

```bash
mvn clean package
```

将生成的 `my-jpostman-plugin-1.0.0.jar` 复制到 JPostman 的插件目录 (`~/.api-client/plugins/loaded/`),或在插件管理界面点击"安装插件"选择 JAR 文件。

## 完整示例: HMAC 签名插件

```java
package com.example;

import com.jpostman.plugin.JPostmanPlugin;
import com.jpostman.plugin.RequestInterceptor;
import com.jpostman.model.KeyValue;
import com.jpostman.model.dto.RequestConfig;

import javax.crypto.Mac;
import javax.crypto.spec.SecretKeySpec;
import java.nio.charset.StandardCharsets;
import java.util.Base64;
import java.util.List;

public class HmacSignPlugin implements JPostmanPlugin, RequestInterceptor {

    private static final String SECRET = "my-secret-key";

    @Override
    public void onLoad() {
        // 初始化资源
    }

    @Override
    public void onUnload() {
        // 释放资源
    }

    @Override
    public String getName() {
        return "HMAC Sign";
    }

    @Override
    public String getVersion() {
        return "1.0.0";
    }

    @Override
    public String getDescription() {
        return "为请求自动添加 HMAC-SHA256 签名 (X-Signature 头)";
    }

    @Override
    public RequestConfig intercept(RequestConfig config) {
        String timestamp = String.valueOf(System.currentTimeMillis() / 1000);
        String dataToSign = config.getMethod() + config.getUrl() + timestamp;
        String signature = hmacSha256(SECRET, dataToSign);

        List<KeyValue> headers = config.getHeaders();
        headers.add(new KeyValue("X-Timestamp", timestamp, "HMAC签名", true));
        headers.add(new KeyValue("X-Signature", signature, "HMAC签名", true));

        return config;
    }

    private String hmacSha256(String key, String data) {
        try {
            Mac mac = Mac.getInstance("HmacSHA256");
            SecretKeySpec secretKey = new SecretKeySpec(
                key.getBytes(StandardCharsets.UTF_8), "HmacSHA256");
            mac.init(secretKey);
            byte[] hash = mac.doFinal(data.getBytes(StandardCharsets.UTF_8));
            return Base64.getEncoder().encodeToString(hash);
        } catch (Exception e) {
            throw new RuntimeException("HMAC 计算失败", e);
        }
    }
}
```

## 扩展点详解

### RequestInterceptor (请求拦截器)

在请求发送前被调用,可修改请求的任何部分 (URL、Headers、Body、Auth)。

```java
public class CustomAuthPlugin implements JPostmanPlugin, RequestInterceptor {
    @Override
    public RequestConfig intercept(RequestConfig config) {
        // 修改 URL
        config.setUrl(config.getUrl() + "?from=plugin");

        // 修改 Body
        config.setBodyType("raw");
        config.setBodyContent("{\"plugin\":\"modified\"}");
        config.setRawContentType("application/json");

        return config;
    }
    // ... 其他方法省略
}
```

### ResponseProcessor (响应处理器)

在收到响应后被调用,可读取或转换响应数据。

```java
public class ResponseLogPlugin implements JPostmanPlugin, ResponseProcessor {
    @Override
    public ResponseData process(ResponseData response) {
        // 记录响应状态
        System.out.println("收到响应: " + response.getStatusCode());

        // 可修改响应体 (如解密)
        if (response.getResponseBody() != null) {
            String decrypted = decrypt(response.getResponseBody());
            response.setResponseBody(decrypted);
        }

        return response;
    }
    // ... 其他方法省略
}
```

### VariableFunction (变量函数)

注册自定义动态变量,在请求中用 `{{$funcName}}` 引用。

```java
public class UuidFunctionPlugin implements JPostmanPlugin, VariableFunction {
    @Override
    public String getFunctionName() {
        return "uuid";
    }

    @Override
    public String execute() {
        return java.util.UUID.randomUUID().toString();
    }

    // JPostmanPlugin 方法...
}
```

在请求 URL 或 Body 中使用: `https://api.example.com/items/{{$uuid}}`

## 调试技巧

- 在 `onLoad()` 中打印日志确认插件被加载
- 插件运行在独立 ClassLoader 中,第三方依赖需打包进 JAR (maven-assembly-plugin 或 maven-shade-plugin)
- 卸载后 ClassLoader 会被关闭,插件中开启的线程/资源需在 `onUnload()` 中释放
- 插件异常不会导致 JPostman 崩溃,会被捕获并记录日志

## 注意事项

- 插件编译目标建议为 Java 8,确保在 JRE 8 上可运行
- `provided` scope 引用 JPostman 接口,避免打包时重复引入
- 一个 JAR 可包含多个插件类,在 SPI 文件中每行写一个实现类
- 插件间通过 ServiceLoader 按顺序加载,执行顺序与 SPI 文件中的排列顺序一致
